# databricks_disaster_recovery_stable_url Resource
[![Public Preview](https://img.shields.io/badge/Release_Stage-Public_Preview-yellowgreen)](https://docs.databricks.com/aws/en/release-notes/release-types)

[API Documentation](https://docs.databricks.com/api/account/disasterrecovery)

A stable URL is a region-independent, failover-aware endpoint for accessing a workspace. It always routes to whichever workspace is currently the primary, so downstream tools and clients do not need to change their connection strings when a failover happens.

The URL is generated by the backend at creation time and is immutable thereafter. A stable URL can be attached to a failover group (via the failover group's `workspace_sets.stable_url_names`) so that its routing target follows the group's primary across failovers.

-> **Note** This resource can only be used with an account-level provider!


## Example Usage
Creating a stable URL bound to an initial workspace:

```hcl
resource "databricks_disaster_recovery_stable_url" "this" {
  stable_url_id        = "accounting-stable-url"
  initial_workspace_id = "1234567890123456"
}
```


## Arguments
The following arguments are supported:
* `initial_workspace_id` (string, required) - The workspace this stable URL is initially bound to. Used only in Create
  requests to associate the stable URL with a workspace. Not returned in
  responses
* `parent` (string, required) - The parent resource. Format: accounts/{account_id}
* `stable_url_id` (string, required) - Client-provided identifier for the stable URL. Used to construct the
  resource name as {parent}/stable-urls/{stable_url_id}

## Attributes
In addition to the above arguments, the following attributes are exported:
* `failover_group_name` (string) - Fully qualified resource name of the FailoverGroup this stable URL is
  currently linked to, in the format
  `accounts/{account_id}/failover-groups/{failover_group_id}`. Empty when
  the stable URL is not attached to any failover group
* `name` (string) - Fully qualified resource name.
  Format: accounts/{account_id}/stable-urls/{stable_url_id}
* `url` (string) - The stable URL endpoint. Generated on creation and
  immutable thereafter. For non-Private-Link workspaces this is
  `https://<spog_host>/?w=<connection_id>`. For Private-Link workspaces
  this is the per-connection hostname

## Import
As of Terraform v1.5, resources can be imported through configuration.
```hcl
import {
  id = "name"
  to = databricks_disaster_recovery_stable_url.this
}
```

If you are using an older version of Terraform, import the resource using the `terraform import` command as follows:
```sh
terraform import databricks_disaster_recovery_stable_url.this "name"
```
